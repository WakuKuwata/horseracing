# Feature Specification: 予測根拠表示 (Prediction Explanation Display)

**Feature Branch**: `040-prediction-explanation`

**Created**: 2026-07-02

**Status**: Draft

**Input**: User description: "予測根拠表示（prediction explanation display） — 各馬の予測根拠（特徴量寄与 top-K）・モデルのグローバル特徴量重要度・モデルと市場の乖離バッジ（vault の「危険な人気馬」「妙味のある穴馬」要求の中立化）を RaceFront に表示する。Obsidian vault open-decisions P2「予測根拠」+ アーキテクチャ詳細ページ表示要件の最後の未実装項目。モデル・特徴量・確率パイプラインには一切介入しない（表示専用、021 と同型）。"

## 背景と目的

製品目的は「人間が予測・確率・期待値とその信頼性を見て自分で判断する正直な意思決定支援」（Feature 021 で確立）。現状 RaceDetailPage はモデル p / 市場 q / 校正 reliability を表示するが、**「なぜこの馬の p が高い/低いのか」の根拠が一切なく**、ユーザーはモデルを盲信するか無視するかしか選べない。プロジェクト元ドキュメント（Obsidian vault）の詳細ページ表示要件のうち唯一「予測根拠」だけが未実装であり（open-decisions P2「予測根拠」）、本 feature がそれを埋める。

3 つの独立した表示を追加する: (1) 各馬の予測根拠（特徴量寄与 top-K）、(2) モデルのグローバル特徴量重要度、(3) モデルと市場の乖離バッジ（vault の「危険な人気馬」「妙味のある穴馬」要求を 021 の中立提示原則で再解釈したもの）。モデル・特徴量・確率パイプラインには一切介入しない（表示専用、021 と同型）。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 各馬の予測根拠（特徴量寄与 top-K） (Priority: P1)

レース詳細画面を見ているユーザーが、ある馬の予測勝率について「モデルは何を根拠にこの値を出したのか」を知りたい。馬の行を展開すると、その馬のスコアを押し上げた/押し下げた要因の上位（既定 5 件）が、日本語の特徴ラベル・実際の値・寄与の向きと大きさ（正負色分けバー）で表示される。表示には「これは校正・レース内正規化前のスコアへの寄与であり、最終確率の内訳ではない」「相関に基づく説明であり因果ではない」という限界注記が常に付く。

**Why this priority**: 本 feature の核。vault 表示要件の最後の未実装項目であり、「盲信か無視か」の二択を「根拠を見て自分で判断」に変える意思決定支援の本丸。

**Independent Test**: 予測根拠付きの予測を生成 → レース詳細画面で馬行を展開 → 寄与 top-K が日本語ラベル・値・正負付きで表示され、限界注記が見えることを確認。根拠が未生成の古い予測では「未提供」と表示される。

**Acceptance Scenarios**:

1. **Given** 根拠付き予測が保存されたレース, **When** ユーザーが馬行を展開する, **Then** 寄与絶対値上位 K 件（既定 5）が日本語ラベル・特徴値・寄与の正負と大きさ付きで表示され、残り特徴の寄与は「その他」として合算表示される
2. **Given** 同じ画面, **When** 根拠を表示する, **Then** 「校正・レース内正規化前のスコア寄与」「因果ではない」の限界注記が同時に表示される
3. **Given** 根拠が保存されていない過去の予測（旧 run）, **When** 馬行を展開する, **Then** 「未提供」が表示される（エラーや空白ではない）
4. **Given** モデル内部生成の特徴（騎手/調教師の成績エンコーディング等）が上位に来る場合, **When** 根拠を表示する, **Then** 内部列名でなく日本語ラベル（例「騎手成績（統計）」）で表示される。日本語ラベル未定義の特徴は元の名前のまま表示される（隠さない）

---

### User Story 2 - モデルのグローバル特徴量重要度 (Priority: P2)

ユーザー（またはモデルの挙動を監査する運用者）が「このモデルは全体としてどの特徴を重視しているのか」を知りたい。モデル情報として、学習済みモデルの特徴量重要度の上位 N 件が日本語ラベル付き横棒チャートで表示される。重要度が記録されていないモデルでは「未収録」の旨が明示される。

**Why this priority**: per-horse 根拠（US1）の背景情報としてモデル全体の傾向を提示し、単一レースの寄与の解釈を助ける。US1 と独立に価値がある（モデル監査用途）。

**Independent Test**: 重要度が記録されたモデルの重要度表示を開く → 上位特徴が日本語ラベル付きで表示される。未記録モデルでは「未収録」として区別表示される。

**Acceptance Scenarios**:

1. **Given** 重要度が記録された学習済みモデル, **When** ユーザーがモデルの重要度表示を開く, **Then** 重要度上位 N 件が日本語ラベル・相対値付きで表示される
2. **Given** 重要度が記録されていない旧モデル, **When** 同じ表示を開く, **Then** 「このモデルには重要度が収録されていない」旨の型付きの未収録状態が表示される（エラー画面ではない）

---

### User Story 3 - モデルと市場の乖離バッジ (Priority: P3)

ユーザーがレース詳細で各馬について「モデルの見立てと市場（オッズ）の見立てがどれだけ食い違うか」を一目で知りたい。各馬に、事前登録した乖離バンドに基づく純事実比較バッジ（「市場評価がモデルより高い」「モデル評価が市場より高い」「ほぼ同等」）が表示される。vault の「危険な人気馬」（市場人気だがモデルは弱気）「妙味のある穴馬」（市場軽視だがモデルは強気）に対応する情報を、**買い推奨と誤読されない中立表現**で提供する。バッジには「モデルと市場の意見相違であり、的中や利益の保証ではない」の注記が付く。

**Why this priority**: p と q は既に併記されている（021）が、数値比較の解釈をユーザー任せにしている。バッジは注意を向ける補助であり、US1/US2 なしでも成立するが本 feature の中では補助的。

**Independent Test**: p と q が揃った馬にバッジが表示され、q 欠損または母集団不一致の馬ではバッジが表示されない（抑制）ことを確認。

**Acceptance Scenarios**:

1. **Given** モデル p と市場 q が同一母集団で揃った馬, **When** レース詳細を表示する, **Then** 事前登録バンドに従った純事実比較バッジ（「市場評価がモデルより高い」「モデル評価が市場より高い」「ほぼ同等」）が表示され、断定・方向性語（危険/妙味/買い/儲かる/弱気/強気 等）は使われない
2. **Given** q が欠損した馬、または母集団不一致（canonical_consistent=false）のレース, **When** 同画面を表示する, **Then** 乖離バッジは表示されない（誤った乖離を出すより出さない）
3. **Given** バッジが表示された馬, **When** ユーザーがバッジの説明を見る, **Then** 「意見相違であり的中・利益の保証ではない」注記が表示される
4. **Given** 画面のどの操作でも, **When** 並び替え等を行う, **Then** 乖離量による並び替え・強調ランキングは提供されない（021 の edge ソート禁止を維持）

---

### Edge Cases

- **根拠と確率の不整合誤読**: 寄与はスコア空間（校正・レース内正規化前）で加法的であり、校正・レース内 softmax・正規化を通った最終確率の内訳ではない → 限界注記を常時表示し、寄与から確率を再構成できるかのような表示（寄与の%表示等）はしない。
- **旧予測（根拠なし）**: 根拠が空の行は「未提供」表示。エラーにしない。backfill はしない（再予測すれば自然に埋まる）。
- **モデル内部特徴の露出**: 学習器内部で生成される特徴（成績エンコーディング列等）が寄与上位に来る → 日本語ラベル対応表で変換。未知の特徴名は fail-open でそのまま表示（隠すと監査性を損なう）。
- **同点・僅差の寄与**: top-K 選択は寄与絶対値降順 + 特徴名昇順の決定論的タイブレーク（再現性、憲法 V）。
- **取消馬**: 予測対象外の馬（取消・除外）には根拠もバッジも表示されない（そもそも p が無い）。
- **q だけある馬 / p だけある馬**: 乖離バッジは p と q が両方有効な馬にのみ表示。片方欠損では抑制。
- **極端に小さい p, q**: 乖離バンドは単純な比率でなく事前登録した相対・絶対の複合条件で判定し、p, q が微小な領域で比率が発散してバッジが乱発されないようにする。
- **モデル更新**: 重要度・根拠は予測 run / モデル版に紐づく。画面には常に「どの run / どのモデルの根拠か」の監査情報（既存の RunAudit 表示）が併記される。

## Requirements *(mandatory)*

### Functional Requirements

#### 各馬の予測根拠（US1）

- **FR-001**: システムは予測生成時に、各（レース, 馬）についてモデルスコアへの特徴量寄与（加法的分解: 全寄与 + ベース値 = スコア）を計算できなければならない。寄与の計算は学習済みモデルと予測時に構築済みの特徴行のみを入力とし、オッズ・当該レース結果を一切使わない。
- **FR-002**: システムは寄与絶対値上位 K 件（既定 K=5、設定可能）を、特徴名・特徴値・寄与値の組として予測行と共に永続化しなければならない。上位 K 件以外の寄与は合算値（「その他」）として保存し、あわせて監査に必要な付帯情報 — ベース値・分解対象のモデルスコア・分解手法の識別子と版・K — を保存し、**保存内容単体から加法性（ベース + top-K + その他 = スコア）が検証可能**でなければならない（codex R2）。保存形式は検証可能な固定スキーマとし、自由形式にしない。
- **FR-003**: 予測の読み出し系は、保存済みの根拠をそのまま提供しなければならない（読み時にモデルを再実行しない）。根拠が保存されていない予測行では「未提供」を型として区別して返す。
- **FR-004**: 画面は馬行の展開操作で寄与 top-K を正負色分けバー・日本語ラベル・特徴値付きで表示し、以下の限界注記を常時併記しなければならない: (a)「校正・レース内正規化前のスコア寄与」、(b)「相関に基づく説明であり因果ではない」。UI の主要語は「予測の理由/根拠」でなく**「スコア寄与」「モデルスコアの要因」**とし、因果説明と読ませる文言を避ける（codex R4）。
- **FR-005**: 特徴名→日本語ラベルの対応は単一の対応表で管理し、モデル内部生成特徴（成績エンコーディング列等）には内部列名を露出しない日本語ラベルを与える。**モデル内部生成特徴（統計エンコーディング列）には「導出特徴」の補助バッジを併記**し、生の観測値と区別する（codex R4）。対応表に無い特徴名はそのまま表示する（fail-open）。
- **FR-006**: 根拠の追加計算は既存の予測値を一切変えてはならない（根拠付き予測と従来予測とで、同一入力に対する p がバイト一致）。

#### グローバル特徴量重要度（US2）

- **FR-007**: システムは学習時にモデルの特徴量重要度を算出し、モデルのメタデータ（既存のモデル指標記録）に記録しなければならない。新しいデータ保存構造は追加しない。
- **FR-008**: 読み出し系はモデル指定で重要度を提供し、未記録のモデルでは型付きの「未収録」を返す（021 の校正表示と同型）。
- **FR-009**: 画面は重要度上位 N 件（既定 N=20）を日本語ラベル付きの横棒チャートで表示する。表示名は「モデルの分割利得（gain）重要度」のように算出法を限定した名称とし、一般的な「特徴の重要さ」と過大に読ませない（codex: gain は高利得分割特徴へ偏るバイアスを持つ）。

#### モデル-市場乖離バッジ（US3）

- **FR-010**: システムは各馬についてモデル p と市場 q（同一母集団で再正規化済み、021 の機構を再利用）から乖離区分を導出する。区分は本 spec で事前登録した 3 バンド（FR-011）とし、実データの結果を見た後からの閾値変更は行わない（変更する場合は別 feature として事前登録し直す）。
- **FR-011**: 事前登録バンド（p, q は同一母集団の勝率確率、乖離指標 = 差分 p − q に相対フロアを併用）:
  - 「市場評価がモデルより高い」: p < q − max(0.03, 0.5×q)
  - 「モデル評価が市場より高い」: p > q + max(0.03, 0.5×q)
  - 「ほぼ同等」: 上記以外
  - ただし q が欠損、または母集団不一致（canonical_consistent=false）の場合はバッジ自体を抑制する。
- **FR-012**: バッジ文言は**方向性・感情を含まない純事実比較**に限る（codex R3: 「弱気/強気」も市場助言的と判断し不採用）。「危険」「妙味」「買い」「儲かる」「弱気」「強気」等の断定・示唆語、損益色（緑=買い等）、乖離量での並び替え・ランキングは禁止（021 原則の継承）。バッジには「モデルと市場の意見相違であり、的中や利益の保証ではない」注記を付す。
- **FR-012b**: 乖離区分は読み時に最新オッズ由来の q から導出されるため、オッズ更新で同一予測 run のバッジが変わりうる。画面は q の基準時点（odds の as_of、021 で表示済み）をバッジ近傍で参照可能にし、「いつ時点の市場との比較か」を明示する（codex R4: スナップショット非保存は既存方針 V に従い、時点の可視化で対処）。
- **FR-013**: 乖離区分は保存せず読み時に既存の保存値（p, q）から純関数で導出する（データ保存構造の変更なし）。

#### 横断（リーク境界・整合性・監査）

- **FR-014**: 根拠・重要度・乖離区分は表示専用の派生値であり、モデルの入力特徴量に一切戻してはならない（leak-guard テストで機械検証）。乖離バッジで使う q は表示側でのみ p と結合し、確率パイプライン上では p≠q 分離を維持する。
- **FR-015**: 本 feature は win→joint（009）の確率導出・特徴量定義（FEATURE_VERSION）・モデル学習ロジックに一切介入しない。既存モデルの再学習を要求しない（根拠は次回以降の予測から付与）。
- **FR-016**: 読み出し系は引き続き読み取り専用であり、本 feature で書き込み操作を追加しない（予測時の根拠永続化は既存の予測書き込み経路の拡張であり、読み出し系からは行わない）。
- **FR-017**: 保存する根拠には、どのモデル・どの予測 run のものかが既存の監査情報（prediction_run / model_version）から一意に辿れること。

### Key Entities

- **予測根拠 (explanation)**: （レース, 馬, 予測 run）ごとの寄与 top-K。各要素 = {特徴名, 特徴値, 寄与値}。付帯: ベース値、その他寄与の合算、分解対象スコア、分解手法の識別子と版、K（保存単体で加法性検証可能・固定スキーマ）。予測行に付随して保存され、無い場合は「未提供」。モデル・run への紐付けは既存の監査情報から辿る。
- **特徴量重要度 (importance)**: モデルごとの {特徴名: 重要度} 上位集合。モデルのメタデータに記録。
- **乖離区分 (divergence band)**: 馬ごとの「弱気/強気/同等/非表示」の 4 状態。保存されず読み時導出。
- **特徴ラベル対応表 (feature label map)**: 特徴名→日本語表示名の単一対応表。フロントの全根拠・重要度表示が共有。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 根拠付き予測が保存されたレースで、ユーザーは 2 クリック以内に任意の馬の予測根拠（上位 5 要因）に到達できる。
- **SC-002**: 保存された根拠の加法性（ベース値 + 全寄与合算 = モデルスコア）が全馬で許容誤差内（相対 1e-6）に収まることが機械検証できる。
- **SC-003**: 根拠機能の有効化前後で、同一入力に対する予測確率 p が全馬でバイト一致する（予測への副作用ゼロ）。
- **SC-004**: 根拠未保存の予測・重要度未収録のモデル・q 欠損の馬で、エラー画面や空白でなく「未提供/未収録/バッジ非表示」の意図した状態が表示される（3 状態すべてテストで検証）。
- **SC-005**: 乖離バッジの区分が、保存済み p, q から事前登録バンドの定義どおりに再現できる（決定論、境界値テスト含む）。
- **SC-006**: 画面上のすべての根拠・バッジ表示に限界注記（スコア寄与/非因果/意見相違）が付いていることを UI テストで機械検証できる（021 の「pseudo 値にはラベル」不変条件と同型）。
- **SC-007**: 表示派生値（根拠・重要度・乖離区分）がモデル入力特徴に含まれないことが leak-guard テストで機械検証できる。

## Assumptions

- 予測根拠の計算は、現行モデル（勾配ブースティング木）が備える加法的寄与分解（TreeSHAP 相当）で行える。カスタム目的関数（039 cond_logit）は学習時に木が何を学ぶかを変えるだけで、予測時の寄与分解は学習済み木集合に対して目的関数非依存に動く（codex R1 同意）— **ただし実装前に、serving が実際に使う booster・iteration・特徴順序で「寄与合算 + ベース = 生スコア」の再構成検証を最初に行い、成立しない場合は本 feature を中断して原因究明する**（039 の eval-spike 教訓、codex R1 の具体レシピ）。
- 寄与計算の予測時レイテンシは本実装前に実測し、許容範囲（1 レース分の予測時間の有意な悪化なし）であることを確認する（codex 追加リスク）。
- 根拠の永続化には予測行への nullable な構造化列の追加（migration 1 本）が必要になる見込み。正当化: 読み出し系は読み取り専用かつ ML 非依存のため読み時計算が不可能であり、予測時に保存する以外の経路がない。旧行は NULL のまま（backfill 不要）。top-K 切り詰め + その他合算で行サイズを有界に保つ。詳細設計は plan で確定（憲法 VI）。
- 重要度はモデルの既存メタデータ記録（学習時に書く指標記録）に追記でき、新しい保存構造は不要。
- 乖離バッジの q は 021 で実装済みの市場勝率（同一母集団再正規化）をそのまま使う。オッズが取り込まれていないレースではバッジは出ない（それで正しい）。
- K=5 / N=20 は表示上の既定値であり、性能・視認性の都合で plan 段階の調整を許す（乖離バンド閾値と違い、結果を見て動かしてよい類の値）。
- 本 feature は表示のみで OOS 採否ゲートの対象外（021 前例）。代わりに SC-002〜007 の機械検証可能な不変条件をテスト化する。
