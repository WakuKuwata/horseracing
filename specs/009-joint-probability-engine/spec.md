# Feature Specification: 結合確率エンジン

**Feature Branch**: `009-joint-probability-engine`

**Created**: 2026-06-23

**Status**: Draft

**Input**: User description: "結合確率エンジン。各馬の単勝確率(Feature 006)から Plackett-Luce/Harville で JRA 全券種の的中確率を導出。整合性保証。exotic オッズ/EV は対象外(将来)。"

## 概要

Feature 006 が出力する各馬の単勝確率(`race_predictions.win_prob`、または in-memory)を入力に、
**Plackett-Luce / Harville(復元なし逐次サンプリング)** で JRA 全券種の的中確率を導出する確率エンジン。
対象券種: 単勝(win)・複勝(place)・馬連(quinella)・馬単(exacta)・ワイド(wide)・三連複(trio)・三連単(trifecta)。
出力は対象レースの各組み合わせの確率。**確率エンジンそのもの**が成果物で、exotic オッズ取得・推定オッズ変換・
exotic EV/推奨は**対象外**(別 P0、将来フィーチャー)。スキーマ変更なし(計算ライブラリ + 評価)。

「利用者」は人間ではなく、確率を消費する将来の exotic 推奨ロジックと、整合性/校正を検証するオペレーター。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 単勝確率から全券種の的中確率を整合的に導出 (Priority: P1) 🎯 MVP

オペレーター(または将来の推奨)が、あるレースの各馬の単勝確率を与えると、全券種の組み合わせ的中確率が
**整合性保証付き**で得られる。

**Why this priority**: 本フィーチャーの中核。憲法 P0「結合確率エンジン」。これ単体で exotic 確率の基盤が完成し、
将来の exotic EV/推奨の入力になる。

**Independent Test**: 既知の単勝確率(N=3/4、一様含む)に対し、馬単/三連単/馬連/三連複/ワイド/複勝の値が
手計算 golden と一致し、整合性不変条件(総和・順序=無順序・周辺一致・範囲)をすべて満たす(合成データ、計算のみ)。

**Acceptance Scenarios**:

1. **Given** 正規化済み単勝確率, **When** 導出, **Then** `Σ_{i≠j} P(馬単 i→j)=1`・`Σ_{順序付き3つ組} P(三連単)=1`
   が許容内で成立する。
2. **Given** 馬単確率, **When** 馬連を計算, **Then** `P(馬連{i,j})=P(馬単 i→j)+P(馬単 j→i)`。
3. **Given** 三連単確率, **When** 各馬の周辺(top3 包含)を集計, **Then** `harville_topk` の top3 と許容内で一致。
   同様に馬単周辺 = top2 と一致(非退化入力)。
4. **Given** ワイド, **When** 馬連と比較, **Then** `P(ワイド{i,j}) >= P(馬連{i,j})`(ワイドは top3 内 2 頭、馬連は top2)。
5. **Given** 取消・除外を含む母集団, **When** 導出, **Then** 除外馬を母集団から外し残存馬の単勝確率を Σ=1 に
   **再正規化してから**派生を計算する(憲法 IV)。取消馬の確率は 0(対象外)。
6. **Given** 端点(p_i→1 や 1−p_i−p_j→0), **When** 導出, **Then** clip/floor で数値安定に処理し、範囲逸脱や
   ゼロ割を起こさない。
7. **Given** 同一入力, **When** 2 回導出, **Then** 完全一致(決定論)。

### User Story 2 - 結合確率の校正を過去データで評価できる (Priority: P1)

オペレーターが期間を指定すると、エンジンの結合確率が**実際の的中頻度に整合するか**を評価でき、誤った独立積近似を
baseline として比較できる。

**Why this priority**: 憲法 III(評価先行)。確率方式の採否は校正評価なしに判断できない。Plackett-Luce が素朴な
独立積より妥当であることを示す。

**Independent Test**: 合成/過去データで、エンジンの組み合わせ確率に対する実現結果の NLL/Brier が、独立積 baseline
(`∝ p_i·p_j` を Σ=1 に再正規化)と同一レース集合・同一条件で比較され、エンジンが baseline 以上に校正される。

**Acceptance Scenarios**:

1. **Given** 過去レースの単勝確率と確定結果, **When** 評価, **Then** 馬単/三連単の実現組み合わせに対する NLL/Brier が
   計測される。
2. **Given** エンジンと独立積 baseline, **When** 同一レース集合で比較, **Then** 両者の校正指標が並び、エンジンが
   baseline を悪化させない(理論的に正しい PL の優位を確認)。
3. **Given** 結果・オッズ, **When** 確率導出, **Then** 確率導出は結果/オッズを参照しない(評価採点のみが結果を使う、
   リーク境界)。

---

### User Story 3 - レースの組み合わせ確率を CLI で表示 (Priority: P2)

オペレーターが prediction_run/レースを指定すると、各券種の上位組み合わせ確率を表示できる。

**Why this priority**: 運用・点検の利便。MVP(US1/US2)成立後の操作性。

**Independent Test**: prediction_run を指定して各券種の上位 K 組み合わせ確率が表示され、合計や整合性メタ情報が出る。

**Acceptance Scenarios**:

1. **Given** prediction_run, **When** CLI 実行, **Then** 券種ごとに上位 K の組み合わせと確率が表示される。

### Edge Cases

- **複勝の頭数依存**: 5–7 頭=top2 包含、8 頭以上=top3 包含、≤4 頭=複勝なし(JRA 規則)。エンジンはこの規則で複勝確率を返す。
- **小頭数の縮退**: N=2 では三連系は定義されない(空/None)。N<2 は無効。各券種は定義可能な範囲でのみ値を返す。
- **端点 p_i≈1 / 1−Σ≈0**: clip/floor し、PL 分母のゼロ割を防ぐ。`harville_topk` の「分母≤eps を無視」する挙動は
  exotic 本計算に継承しない(clip + 再正規化で質量欠損を防ぐ)。
- **再正規化前の派生禁止**: 取消・除外除去 → 再正規化 → 派生 の順を厳守(順序違反は確率が壊れる)。
- **同着**: 評価採点では、確定着順(同着 1 着等)が券種の的中条件を満たすかを JRA 規則で判定(同着は的中に含める)。
  確率導出は連続的で同着を区別しない(同着確率は実質ゼロ)。
- **大頭数**: N≤18(JRA 上限)。三連単は N(N−1)(N−2)≈4900 で実用範囲。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは各馬の単勝確率から、馬単 `P(i→j)=p_i·p_j/(1−p_i)`、三連単
  `P(i→j→k)=p_i·(p_j/(1−p_i))·(p_k/(1−p_i−p_j))` を **Plackett-Luce(復元なし逐次)** で導出する MUST。
- **FR-002**: システムは無順序券種を順序の和で導出する MUST(馬連`{i,j}`=馬単(i→j)+馬単(j→i)、三連複は 6 順序の和)。
- **FR-003**: システムは複勝/ワイドを **top-N 包含確率**で導出する MUST。**ワイドは top3 内に 2 頭がともに入る確率**で、
  独立積 `top3_i×top3_j` で近似しない(順序付き top3 列挙の和)。複勝は頭数依存(5–7=top2, 8+=top3, ≤4=なし)。
- **FR-004**: 責務分離 — **呼び出し側**(校正評価・CLI)が取消・除外を母集団から除外して**出走馬の単勝確率のみ**を
  エンジンに渡す。**エンジン**は受け取った母集団を Σ=1 に**再正規化してから**(分母計算より必ず先に)clip → 全派生を
  計算する MUST(憲法 IV)。母集団に含まれない取消・除外馬の確率は出力に現れない(=0 相当)。
- **FR-005**: システムは端点を clip/floor して数値安定に処理する MUST(p∈[eps,1−eps]、PL 分母のゼロ割回避)。
  `harville_topk` の分母 skip 挙動を本計算に継承しない。
- **FR-006**: 出力は整合性不変条件を満たす MUST: `Σ馬単=1`・`Σ三連単=1`(許容内)、無順序=順序和、joint の周辺
  (top1/top2/top3)が `harville_topk` と許容内で一致(非退化入力)、複勝/三連複/ワイドの各馬包含確率 `∈[0,1]`、
  `p_i>=p_j ⟹ place_i>=place_j`(単調性)。
- **FR-007**: 導出は決定論的 MUST(同一入力で同一出力)。
- **FR-008**: 確率導出は**レース結果・オッズを参照しない** MUST(入力は単勝確率のみ。リーク境界)。
- **FR-009**: システムは結合確率の校正を過去データで評価する harness を提供する MUST(実現組み合わせに対する
  NLL/Brier)。**独立積 baseline**(`∝ p_i·p_j` 等を Σ=1 に再正規化)と同一レース集合・同一条件で比較する。
- **FR-010**: システムは小頭数/複勝なし等、券種が定義不能な場合に明確に「該当なし」を返す MUST(誤った値を返さない)。
- **FR-011**: システムは CLI で prediction_run/レースを指定し、各券種の上位 K 組み合わせ確率を表示できる MUST
  (`show`)。期間を指定した校正評価の表示(`calibrate`、US2/FR-009 の harness の CLI 面)も提供する。
- **FR-012**: MVP はスキーマ変更なし(計算ライブラリ + 評価)。exotic 確率の永続化は exotic オッズ源が入るまで保留する MUST。

### Key Entities *(include if feature involves data)*

- **WinProbabilities**: 1 レースの各馬(出走母集団)の正規化済み単勝確率(入力)。
- **JointProbabilities**: 1 レースの券種別組み合わせ確率(win/place/quinella/exacta/wide/trio/trifecta)。
- **結合確率エンジン**: WinProbabilities → JointProbabilities(再正規化→clip→PL 派生→整合性)。
- **校正レポート**: エンジン vs 独立積 baseline の NLL/Brier(同一レース集合、疑似でない=結果が確定した過去データ)。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 既知入力(N=3/4・一様)で全券種の確率が手計算 golden と許容内で一致する。
- **SC-002**: `Σ馬単=1`・`Σ三連単=1`、無順序=順序和、joint 周辺=`harville_topk`(非退化)、包含確率∈[0,1]、単調性が
  すべて成立する。
- **SC-003**: 取消・除外を除外し残存馬で再正規化してから派生され、取消馬の確率が 0 になる。
- **SC-004**: 端点入力でゼロ割・範囲逸脱を起こさず、決定論的に同一出力を返す。
- **SC-005**: 結合確率の校正評価が実現結果の NLL/Brier を算出し、独立積 baseline と同一条件で比較され、エンジンが
  baseline を悪化させない。
- **SC-006**: 複勝の頭数依存・小頭数縮退・該当なしが規則どおりに扱われる。
- **SC-007**: 確率導出が結果/オッズを参照しない(リーク境界)。

## Assumptions

- Feature 003(`harville_topk`)・006(単勝確率)が適用済み。入力単勝確率は出走母集団で正規化済み(必要なら再正規化)。
- JRA 券種規則: 複勝 5–7 頭=top2/8+=top3/≤4=なし。同着は評価採点で JRA 規則により的中判定(確率導出は連続)。
- N≤18(JRA 上限)。確率手法は Plackett-Luce/Harville(主モデル)。他手法(共役・コピュラ等)は対象外。
- 許容誤差は spec/実装で定義(整合性検査・周辺一致)。`harville_topk` との一致は非退化入力で評価。
- スキーマ変更なし。exotic オッズ取得・推定オッズ変換・exotic EV/推奨・同着確率モデルは将来フィーチャー。
- 評価は結果が確定した過去データ(疑似評価ではない=確率校正の評価)。オッズは使わない。
