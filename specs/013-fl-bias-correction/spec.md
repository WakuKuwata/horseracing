# Feature Specification: 人気-不人気バイアス補正（favorite-longshot bias correction）

**Feature Branch**: `013-fl-bias-correction`

**Created**: 2026-06-25

**Status**: Draft

**Input**: User description: "市場含意勝率 q=(1/odds)/Σ(1/odds)（投票シェア、人気-不人気バイアス含む）を過去の (単勝オッズ, 着順) から実現勝率に校正し q' を得る。q' を 009 に通し 010/011/012 の推定オッズ・EV の現実性を上げる。train-only/walk-forward、p≠q、オッズは特徴量にしない、評価先行。"

## 概要

Feature 010 の市場含意勝率 `q_i=(1/odds_i)/Σ_j(1/odds_j)` は**投票シェア**であり、真の勝率ではなく**人気-不人気バイアス**
（favorite-longshot bias: 人気馬は過小評価・大穴は過大評価される傾向）を含む。本フィーチャーは、過去の（単勝オッズ →
q）と**実現勝敗（1着か否か）**の関係から校正器を学習し、補正済み市場勝率 `q'` を得る。`q'` を Feature 009 エンジンに通すこと
で、010 の推定市場オッズ・011/012 の EV・実 ROI の現実性を上げる。

**最重要（リーク境界 / p≠q）**: 校正は**過去データのみ（train-only / walk-forward、日時分割）**で学習し、評価・適用対象レースの
結果を一切混入させない（憲法 II）。`q'` は**市場由来**であり、予測モデル確率 `p`（006/009 on モデル）とは**別物**で混同しない。
オッズおよび `q'` は**予測モデルの特徴量に一切使わない**（`q→q'` は市場オッズ側の変換であり、win モデルには戻さない）。

**評価先行（憲法 III）**: (1) 第一指標として **`q` vs `q'` の勝率校正**（NLL/Brier/信頼性曲線、人気帯別）を実現勝敗に対して
計測し、補正が校正を改善するかを直接検証する。(2) 補助指標として Feature 012 の「推定 vs 実 exotic オッズ」乖離ハーネスを
再利用し、補正前後の coverage/`log(実/推定)` MAE/P90 を比較する。推定オッズは引き続き**疑似評価**と明示する。

スキーマ変更なし。`probability` を拡張し、`betting`(011/012) が補正済み推定オッズを opt-in で使えるようにする。

「利用者」は人間ではなく、校正器を学習・評価・適用するオペレーターと、補正済み推定オッズを消費する 011/012、将来の運用 UI。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 過去データから FL バイアス校正器を学習し q→q' を適用 (Priority: P1) 🎯 MVP

オペレーターが学習期間を指定すると、過去の（市場含意勝率 q, 実現勝敗）から FL バイアス校正器が学習され、対象レースの q に
適用して補正済み q'（レース内 Σ=1）が得られる。

**Why this priority**: 補正器の学習・適用が本フィーチャーの土台。これが無ければ補正済み推定オッズも評価もできない。

**Independent Test**: 合成データ（人気馬が過小評価されるよう歪めた q と実現勝率）で校正器を学習し、対象レースの q に適用した
q' が (a) 各馬で単調変換、(b) レース内 Σ=1 に再正規化、(c) 学習に評価対象レースの結果を使っていない（walk-forward）、
(d) 同一入力で決定論、を満たすことを確認。

**Acceptance Scenarios**:

1. **Given** 過去の (q, 実現勝敗) と対象レース, **When** 校正器を学習し適用, **Then** 補正済み `q'` が得られ、学習窓は対象
   レースの**日時より前**に限定（walk-forward、結果リークなし）。
2. **Given** 校正器（単調）, **When** q に適用, **Then** 各馬で `q'_i = f(q_i)` は**単調**（人気→不人気の順序を保持）し、
   レース内で **Σ q' = 1** に再正規化（取消・除外・無効オッズは除外して再正規化）。
3. **Given** 学習データに無いオッズ帯, **When** 適用, **Then** 外挿は単調性を壊さず安定（端点クリップ等）に扱う。
4. **Given** 校正方式（isotonic / べき乗 / log-log）と学習窓, **When** 学習, **Then** 方式・パラメータ・学習窓が `logic_version`
   相当のメタに記録され、再現可能。
5. **Given** モデル確率 p, **When** 補正, **Then** 補正は **q のみ**を入力とし p を一切参照しない（p≠q）。

---

### User Story 2 - 補正済み q' で推定 exotic オッズ・EV を算出 (Priority: P1)

オペレーターが補正を有効化すると、`q'` が Feature 009 エンジンに渡され、010 の推定市場オッズ・011/012 の EV が補正済み q' を
用いて算出される（無効時は従来の生 q）。

**Why this priority**: 補正の価値は推定オッズ・EV の改善にあり、それを 010/011/012 の経路に接続して初めて意味を持つ。

**Independent Test**: 同一レースで「補正なし（生 q）」と「補正あり（q'）」の推定 exotic オッズを算出し、(a) 補正ありが校正済み
q' に基づくこと、(b) 011/012 の EV 経路が補正済み推定オッズを opt-in で使えること、(c) p（モデル側）は不変であること、を確認。

**Acceptance Scenarios**:

1. **Given** 補正有効, **When** 推定オッズ算出, **Then** 010 の `O_est=(1−控除率)/P_market` が **q' 由来の P_market** で計算される。
2. **Given** 補正有効, **When** 011/012 の EV 算出, **Then** `EV=P_model(p)×O_est(q')`（p はモデル、q' は補正済み市場）で、
   p と q' を混同しない。
3. **Given** 単勝オッズ復元, **When** 補正あり, **Then** 補正済み推定単勝オッズは**生オッズを厳密復元しない**（補正の意図的帰結）
   ことを明示する（生 q では復元、q' では乖離＝バイアス除去）。
4. **Given** 補正無効, **When** 算出, **Then** 従来どおり生 q による 010/011/012 の挙動（後方互換）。

### User Story 3 - 補正の効果を評価（勝率校正 + 乖離比較） (Priority: P1)

オペレーターが評価期間を指定すると、補正前後の市場勝率の校正品質と、推定 exotic オッズの実オッズへの乖離が計測され、
補正の採否を判断できる。

**Why this priority**: 憲法 III（評価先行）。補正が校正を改善せず乖離も縮めないなら採用しない。

**Independent Test**: walk-forward の評価期間で、(a) **第一指標** `q` vs `q'` の NLL/Brier/信頼性曲線（人気帯別）を実現勝敗に
対して算出し補正が改善するか、(b) **補助指標** 012 の乖離ハーネスで補正前後の coverage/`log(実/推定)` MAE/P90 を比較、
を確認。全推定値が疑似評価明示。

**Acceptance Scenarios**:

1. **Given** 評価期間（walk-forward）, **When** 評価, **Then** `q` と `q'` の勝率校正（NLL/Brier/信頼性曲線・ECE 相当）が
   **人気帯別**に算出され、補正の改善/悪化が定量化される。
2. **Given** 実 exotic オッズ（012）, **When** 乖離比較, **Then** 補正前（生 q）と補正後（q'）で coverage/`log(実/推定)`
   median/MAE/P90 が並び、補正が乖離を縮めるかが分かる。
3. **Given** 小サンプルの人気帯, **When** 集計, **Then** サンプル数を明示し、過小サンプル帯を誤解釈しない。
4. **Given** 評価出力, **When** レポート, **Then** 推定オッズは**疑似評価**、`q'` は市場由来でモデル p ではないと明示。
5. **Given** 補正のターゲット, **When** 解釈, **Then** 主指標は**実現勝率への校正**（直接の真値）で、乖離（実 exotic 価格には
   独自の控除・偏りがある）は**補助**と位置づける。

### User Story 4 - CLI で校正器の学習・適用・評価 (Priority: P2)

オペレーターが CLI で、校正器の学習（期間・方式指定）、評価（walk-forward）、推奨/推定への適用切替を実行できる。

**Why this priority**: 運用効率。MVP(US1–US3) 成立後の操作性。

**Independent Test**: CLI で校正器学習・評価を実行し、勝率校正指標（人気帯別）と乖離比較が方式・学習窓明示で表示される。

**Acceptance Scenarios**:

1. **Given** 学習期間・方式, **When** 学習 CLI, **Then** 校正器の要約（方式・パラメータ・学習窓・サンプル数）が表示される。
2. **Given** 評価期間, **When** 評価 CLI, **Then** 補正前後の校正指標・乖離指標が疑似評価明示で表示される。

---

### Edge Cases

- **校正対象は「レース正規化後の q'」（CRITICAL）**: 単調写像 `g` を各馬の q_i に適用後、レース内で `q'_i=g(q_i)/Σ_j g(q_j)` に
  再正規化する。**校正の学習・評価はこの正規化後 q' に対して行う**（per-horse の生 `g(q_i)` を真値扱いしない — 再正規化で marginal
  が変わるため）。正準方式は**べき乗モデル `q'_i ∝ q_i^γ`**（γ を勝者尤度 MLE で学習、単調・外挿安定・レース文脈を正規化で考慮）。
  isotonic 等の代替も**正規化後出力で**学習・評価。009 には正規化済みベクトルを渡し、エンジンの再正規化/clip が無作用になるよう整合。
- **オッズ復元の非保存**: 010 SC-001 の「生オッズ復元」は生 q でのみ成立。補正後は意図的に乖離（バイアス除去）し、これを明示。
- **walk-forward**: 学習窓は対象レースの日時より厳密に前。評価期間の結果は学習に使わない（リーク防止）。
- **方式と外挿**: isotonic（単調保証・過学習に強いが学習レンジ外は端点一定）/ パラメトリック（べき乗・log-log、外挿可だが
  形状仮定）。学習レンジ外オッズは端点クリップ等で安定化。方式は設定可能で評価で選ぶ。
- **小サンプル人気帯**: 信頼性曲線・帯別集計はサンプル数併記。過小帯は単調平滑化・統合で扱う。
- **取消・除外・無効オッズ**: q 計算前に除外し、補正・再正規化も残存馬で行う（憲法 IV）。
- **評価ターゲットの偏り**: 実 exotic 価格は独自の控除・偏りを含むため、乖離縮小を唯一の成功条件にしない。主指標は実現勝率校正。
- **後方互換**: 補正は opt-in。無効時は 010/011/012 の従来挙動（生 q）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは過去の（市場含意勝率 q, 実現勝敗=1着か否か）から **FL バイアス校正器** `f: q → q'` を学習する MUST。学習は
  **train-only / walk-forward（日時分割）**で、適用・評価対象レースの結果を一切使わない MUST（憲法 II）。
- **FR-002**: システムは校正器を対象レースの q に適用し、各馬で **単調**な `g(q_i)` を得て、レース内で `q'_i=g(q_i)/Σ_j g(q_j)`
  （**Σ q'=1**）に再正規化する MUST。**校正の学習・評価は正規化後 q' に対して**行い（再正規化が marginal を変えるため per-horse の
  生 `g(q_i)` を真値扱いしない）、正準方式は**べき乗 `q'∝q^γ`**（γ を勝者尤度で学習）とし isotonic 等の代替も正規化後で学習する MUST。
  取消・除外・無効オッズは除外し残存馬で再正規化（field_size は補正後の出走集合から導出）する MUST（憲法 IV）。009 へ渡す q' は
  エンジンの再正規化/clip が無作用になるよう整合させる MUST。
- **FR-003**: システムは補正を **q のみ**を入力として行い、モデル確率 p を一切参照しない MUST（p≠q）。オッズおよび q' を予測
  モデルの特徴量に**一切使わない** MUST（憲法 II）。
- **FR-004**: システムは校正方式を設定可能とし、**MVP は べき乗 `q'∝q^γ`（正準）のみ実装**、isotonic/log-log は将来（正規化後
  conditional-logit 目的でのアルゴリズム実装が必要なため `method` 引数は受けるが未実装は明示エラー）とする MUST。方式・パラメータ・
  学習窓・サンプル数を再現可能なメタ（logic_version 相当）に記録する MUST（憲法 V）。**方式・ハイパーパラメータ（γ 探索範囲等）の
  選択は学習窓内（勝者尤度 MLE / nested walk-forward）で行い、最終評価期間を選択に使わない** MUST（選択リーク防止、憲法 II/III）。
- **FR-005**: システムは補正済み q' を Feature 009 エンジンに渡し、010 の推定市場オッズ・011/012 の EV を **q' 由来**で算出できる
  MUST。補正は **opt-in** で、無効時は生 q による従来挙動（後方互換）。
- **FR-006**: システムは補正後の推定単勝オッズが**生オッズを厳密復元しないこと**（バイアス除去の意図的帰結）を明示する MUST。
- **FR-007**: システムは **第一評価指標**として、walk-forward 評価期間で `q` と `q'` の勝率校正（**NLL/Brier/信頼性曲線/ECE 相当、
  人気帯別**）を実現勝敗に対して算出する MUST。ECE/信頼性は**正規化後 q'** に対し、固定ビン境界・空ビン処理・サンプル数併記を
  定義して計算する MUST。**採否ゲートは勝率校正（NLL/Brier/ECE）**で判断し、乖離（FR-008）は診断補助に留める MUST。
- **FR-008**: システムは **補助評価指標**として、Feature 012 の乖離ハーネスを再利用し、補正前後の coverage/`log(実/推定)`
  median/MAE/P90 を比較する MUST。乖離縮小は補助であり唯一の成功条件にしない。
- **FR-009**: システムは全推定オッズ・評価出力を**疑似評価**として明示し、q' が市場由来でモデル p ではないと区別する MUST。
- **FR-010**: 学習・適用・評価は決定論的 MUST（同一入力・同一方式・同一窓で同一）。小サンプル人気帯はサンプル数を明示する MUST。
- **FR-011**: システムは CLI で、校正器の学習（期間・方式）・評価（walk-forward）・推定/推奨への適用切替を実行できる MUST。
- **FR-012**: 本フィーチャーはスキーマ変更なし。完全な多出力モデル・Kelly・**モデル p 側**のバイアス補正は将来に明示分離する MUST。

### Key Entities *(include if feature involves data)*

- **FLCalibrator**: q → q' の単調写像（方式・パラメータ・学習窓・サンプル数を保持）。市場由来、p 非参照。
- **CorrectedMarketProbs**: レース内で Σ=1 に再正規化した補正済み市場勝率 q'（009 入力）。
- **校正評価レポート**: 人気帯別の NLL/Brier/信頼性曲線/ECE（q vs q'）+ サンプル数（第一指標）。
- **乖離比較レポート**: 補正前後の coverage/log 比 median/MAE/P90（012 ハーネス、補助指標、疑似）。

### Measurable Outcomes

- **SC-001**: 過去データから FL 校正器が walk-forward で学習され、対象レースの q に適用した q' が各馬で単調・レース内 Σ=1、
  学習に評価対象レースの結果を使わない。
- **SC-002**: 補正は q のみを入力とし p を参照せず、オッズ/q' がモデル特徴に使われない（p≠q、リーク境界）。
- **SC-003**: 補正済み q' が 009/010 に渡り、011/012 の EV が q' 由来で算出できる（opt-in、無効時は後方互換）。補正後の推定単勝
  オッズは生オッズを厳密復元しない（明示）。
- **SC-004**: 第一指標として q vs q' の勝率校正（NLL/Brier/信頼性曲線/ECE、人気帯別）が実現勝敗に対して算出され、改善/悪化が

- **SC-005**: 補助指標として補正前後の推定 vs 実 exotic 乖離（coverage/log 比 MAE/P90）が比較される。
- **SC-006**: 校正方式が設定可能で、方式・パラメータ・学習窓・サンプル数が再現メタに記録される。決定論的。

## Assumptions

- Feature 009（結合確率）・010（推定市場オッズ）・011（exotic EV）・012（実 exotic オッズ + 乖離ハーネス）が適用済み。
  `probability` を拡張し `betting` が opt-in 利用する。
- 学習データは `race_horses.odds`（単勝）+ `race_results`（1着か否か）。2007+。walk-forward は**対象レースの開始時刻より厳密に前**
  に限定し、同日同時刻は race_id で決定論タイブレーク（日付単位 `<=` は使わない）。
- **オッズの性質（限界の明示）**: `race_horses.odds` は確定/締切寄り（closing-leaning、008 は result-pending のみ上書き、JRA-VAN
  確定を保護）。本フィーチャーは**回顧的（retrospective）な市場校正研究**としてこの odds で学習・評価する。**運用（operational）**
  では出走前オッズ（008/012）を使うべきで、締切オッズで学習した校正が朝オッズへ完全転移する保証はない（既存 007/010 の
  closing-oracle 簡略化と同じ前提・限界）。post-start オッズを deployable EV の入力にしない。
- 同着（複数 1 着）は勝率校正のサンプルから除外し件数を明示（既存 market_calibration と同方針）。
- 校正は**単勝市場のべき**（q を補正）に適用し、補正済み q を 009 に通すことで exotic も整合的に補正される（exotic 実価格への
  直接フィットはしない）。
- 既定方式は**べき乗 `q'∝q^γ`**（正準、レース正規化後を勝者尤度で学習、単調・外挿安定）。isotonic は per-horse 単調診断の代替で
  学習レンジ外は端点クリップ + 範囲外件数を報告。方式選択は学習窓内で行う（最終評価を選択に使わない）。人気帯は固定境界（popularity
  または q 分位の固定エッジ）で定義し、過小帯はサンプル数明示・統合。
- 補正済み推定オッズは `probability` に**新しい opt-in 経路**として追加（010 の `estimate_market_odds` 生 q 経路は後方互換で維持）。
  q' は市場由来で、`features`/`serving` の win モデル入力に一切流さない（リーク・ガードテストで担保）。
- 実 exotic 価格は独自の控除・偏りを含むため、乖離縮小は補助指標。主指標は実現勝率校正（直接の真値）。
- スキーマ変更なし。モデル p 側補正・Kelly・多出力モデルは将来。日本語規約維持。
