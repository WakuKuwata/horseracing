# Feature Specification: 推定市場オッズ変換

**Feature Branch**: `010-estimated-market-odds`

**Created**: 2026-06-23

**Status**: Draft

**Input**: User description: "推定市場オッズ変換。単勝オッズから各券種の推定市場オッズを導出(市場含意 win 確率 q → Feature 009 → 控除率)。実オッズと区別し『推定』として明示。EV/推奨は将来。"

## 概要

実際の exotic オッズ(馬連/馬単/ワイド/三連複/三連単)が無い/未来レースのため、**単勝オッズ**
(`race_horses.odds`、または 008 でスクレイプした前売り単勝オッズ)から各券種の**推定市場オッズ**を導出する
変換規則(憲法 P0「推定市場オッズ変換規則」)。

変換規則:
1. 単勝オッズ `odds_i` から**市場含意 win 確率** `q_i = (1/odds_i) / Σ_j(1/odds_j)`(オーバーラウンド=控除を
   除去した市場の投票シェア。**真の勝率でもモデル確率 p でもない**)。
2. この `q`(モデル確率 p ではない)を Feature 009 の結合確率エンジン(Plackett-Luce)に入力し、各券種の
   **市場含意的中確率** `P_market(c)` を得る。
3. 券種別の**控除率**(takeout)を適用し、**推定オッズ** `= (1 − takeout_券種) / P_market(c)`。

成果物は推定オッズの導出までで、**EV/推奨は対象外**(将来 betting で「モデル確率 p × 推定オッズ」)。スキーマ変更
なし(計算ライブラリ + 評価)。

**重要な区別**: 推定に使う win 確率は**市場由来 q**、予測は**モデル p**(Feature 006)。両者を混同しない。推定オッズは
実オッズではないため**「推定(is_estimated_odds)」として明示**し、実オッズ由来と区別する(憲法 V)。実 exotic 市場は
券種別の独立投票プールで決まるため、Plackett-Luce 外挿の推定オッズは実 exotic オッズと**乖離しうる**(疑似評価)。

「利用者」は人間ではなく、推定オッズを消費する将来の exotic EV/推奨と、変換規則を検証するオペレーター。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 単勝オッズから各券種の推定市場オッズを導出 (Priority: P1) 🎯 MVP

オペレーター(または将来の EV)が、あるレースの単勝オッズを与えると、各券種の推定市場オッズが得られる。

**Why this priority**: 本フィーチャーの中核。憲法 P0「推定市場オッズ変換規則」。これ単体で exotic オッズの推定基盤が
完成し、009 のモデル確率と組み合わせた exotic EV の前提になる。

**Independent Test**: 人工オッズ `odds_i = R/s_i`(投票シェア s、払戻率 R)で `q_i = s_i`、推定単勝オッズ
`= R/q_i = odds_i` を厳密復元。市場含意 `q` を 009 に通した各券種の推定オッズが整合的(`q` 入力でも Σ=1 等)に
得られる(合成データ、計算のみ)。

**Acceptance Scenarios**:

1. **Given** 単勝オッズ, **When** 市場含意 q を計算, **Then** `q_i = (1/odds_i)/Σ(1/odds_j)`、`Σq=1`。`q` は市場の
   投票シェア(真の勝率/モデル p ではない)として扱われる。
2. **Given** 市場含意 q, **When** 009 エンジンに入力, **Then** 各券種の市場含意確率が得られ、`Σ馬単=1`・`Σ三連単=1`・
   `wide=Σ_k trio` 等の整合性を満たす。
3. **Given** 市場含意確率と控除率, **When** 推定オッズを計算, **Then** `推定オッズ = (1−takeout_券種)/P_market(c)`。
   控除率は券種別(既定: 単勝/複勝 20%・馬連/ワイド 22.5%・馬単/三連複 25%・三連単 27.5%)で設定可能。
4. **Given** 控除率が実オーバーラウンドに一致(`Σ1/odds=1/(1−takeout_win)`), **When** 単勝に変換規則を適用,
   **Then** 推定単勝オッズが実 `odds_i` を厳密復元する。
5. **Given** 推定確率 `P_market(c)→0`, **When** 推定オッズ, **Then** 派生値の推定オッズを上限 cap/None で扱う
   (**確率本体は cap しない**=整合性を壊さない)。

---

### User Story 2 - 変換規則を過去データで検証 (Priority: P1)

オペレーターが期間を指定すると、変換規則の妥当性(単勝オッズ復元・市場含意 q の校正)を過去データで検証できる。

**Why this priority**: 憲法 III(評価先行)。推定規則は検証なしに採否を判断できない。実 exotic オッズが無いため、
検証可能な範囲(単勝復元・q 校正・実現組み合わせの結果校正)で妥当性を示す。

**Independent Test**: 過去レースで、推定単勝オッズ vs 実 `odds_i` の復元誤差(レース単位 `|log(R·S_r)|` 等)を計測し、
市場含意 q の校正(実勝馬に対する NLL/Brier)を評価する。全評価は**疑似評価**(推定オッズは実 exotic 価格ではない)。

**Acceptance Scenarios**:

1. **Given** 過去レースの単勝オッズ, **When** 単勝復元を評価, **Then** レース単位の復元誤差(`R_win·S_r` の log 等)が
   計測される。控除率=実オーバーラウンドなら誤差≈0。
2. **Given** 市場含意 q と確定結果, **When** 校正評価, **Then** 実勝馬に対する q の NLL/Brier が計測される(市場の
   予測力の参照)。
3. **Given** 評価出力, **When** レポート, **Then** **疑似評価(推定市場オッズ、実 exotic 価格ではない)**と明示される。

---

### User Story 3 - レースの推定オッズを CLI で表示 (Priority: P2)

オペレーターがレースを指定すると、各券種の推定市場オッズ(上位)を表示できる。

**Why this priority**: 運用・点検の利便。MVP(US1/US2)成立後の操作性。

**Independent Test**: レース指定で各券種の推定オッズ上位 K と控除率・疑似明示が表示される。

**Acceptance Scenarios**:

1. **Given** レース, **When** CLI 実行, **Then** 券種別の推定オッズ上位 K と使用控除率・**「推定」明示**が表示される。

---

### Edge Cases

- **オッズ欠損/0/負**: その馬を市場含意 q の母集団から除外する(価格付け不能)。除外後に q を再正規化。残存が少なすぎる/
  Σ1/odds≤0 なら推定不能を明示。
- **取消・除外**: 出走母集団から除外して q を再正規化(憲法 IV、009 の呼び出し側責務と同じ)。
- **推定確率 0 近傍**: 推定オッズ(派生値)を上限 cap または None。**確率本体は cap しない**(Σ=1 等を壊さない)。
- **複勝の推定**: 頭数依存(5–7=top2/8+=top3/≤4=なし)。ただし複勝の実払戻は払戻対象頭数とプール分配で変動するため、
  `(1−takeout)/P(place)` は**粗い近似**であることを明示。
- **控除率の時点依存**: JRA 控除率は改定されうる(既定値は平成26年6月7日以降)。控除率は設定可能で logic_version に含める。
- **小頭数**: 三連系は N<3 で該当なし。各券種は定義可能な範囲でのみ推定。
- **closing-oracle**: 確定単勝オッズ(結果時点)を使う場合は疑似評価。前売りオッズ(008)はより実運用寄り。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは単勝オッズから市場含意 win 確率 `q_i = (1/odds_i)/Σ_j(1/odds_j)` を計算する MUST。`q` は
  市場の投票シェアであり、**真の勝率でもモデル確率 p でもない**(命名・保存・メタで区別)。
- **FR-002**: システムは市場含意 `q` を Feature 009 の結合確率エンジンに入力して各券種の市場含意的中確率を導出する
  MUST(`p` ではなく `q` を入力)。009 の整合性(Σ=1・無順序=順序和・wide=Σ_k trio)を満たす。
- **FR-003**: システムは券種別控除率を適用し `推定オッズ = (1 − takeout_券種)/P_market(c)` を計算する MUST。控除率は
  券種別に設定可能(既定: 単勝/複勝 20%・馬連/ワイド 22.5%・馬単/三連複 25%・三連単 27.5%)で、logic_version に含める。
- **FR-004**: システムはオッズ欠損/0/負の馬を q 母集団から除外し、残存馬で q を再正規化する MUST。取消・除外も母集団
  から除外する(憲法 IV)。推定不能(残存不足/Σ1/odds≤0)を明確に返す。
- **FR-005**: システムは推定確率が 0 近傍のとき**推定オッズ(派生値)を上限 cap/None** で扱う MUST。**確率本体は cap
  しない**(整合性を壊さない)。
- **FR-006**: システムは推定単勝オッズが、控除率=実オーバーラウンドのとき実 `odds_i` を復元する性質を満たす MUST
  (`推定 = R·S·odds_i`、`S=Σ1/odds`)。
- **FR-007**: 推定オッズは**実オッズではないため「推定(is_estimated_odds)」として明示**する MUST。実オッズ由来値と
  区別し、実 exotic 価格との乖離(Plackett-Luce 外挿)を疑似評価として扱う(憲法 V)。
- **FR-008**: 変換は**予測モデル確率 p を一切使わない** MUST(入力は市場オッズのみ)。p と q を同一オブジェクト/列に
  混入させない。オッズは予測モデルの特徴ではない(005/006 で担保、本フィーチャーは市場側)。
- **FR-009**: システムは過去データで(a)単勝オッズ復元誤差(レース単位)、(b)市場含意 q の校正(実勝馬に対する
  NLL/Brier)を評価する harness を提供する MUST。全評価出力を**疑似評価**として明示する。
- **FR-010**: 変換は決定論的 MUST(同一入力で同一出力)。複勝の頭数依存・小頭数を規則どおり扱う。
- **FR-011**: システムは CLI でレースを指定し、各券種の推定オッズ上位 K と使用控除率・推定明示を表示できる MUST。
- **FR-012**: MVP はスキーマ変更なし(計算ライブラリ + 評価)。推定オッズの永続化は betting 統合まで保留する MUST
  (将来 `recommendations.estimated_market_odds_used`/`is_estimated_odds` に入る)。

### Key Entities *(include if feature involves data)*

- **WinOdds**: 1 レースの各馬の単勝オッズ(入力。欠損/0/負は除外)。
- **MarketImpliedWinProbs (q)**: オーバーラウンド除去後の市場含意 win 確率(投票シェア)。モデル確率 p とは別物。
- **EstimatedOdds**: 券種別の推定市場オッズ(`(1−takeout)/P_market`)。**推定フラグ付き**、実オッズと区別。
- **推定市場オッズ変換器**: WinOdds → q → (009) → P_market → EstimatedOdds。控除率設定可能。
- **検証レポート**: 単勝復元誤差・q 校正(NLL/Brier)。疑似評価。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 人工オッズ `odds_i=R/s_i` で `q_i=s_i`、推定単勝オッズ `=odds_i` を厳密復元(誤差 0、許容内)。
- **SC-002**: 市場含意 q を 009 に通した各券種の推定オッズが整合的に得られ(Σ=1 等)、控除率で `(1−takeout)/P` になる。
- **SC-003**: オッズ欠損/0/負・取消・除外を除外して q を再正規化し、推定不能を明確に返す。推定確率 0 近傍で推定オッズが
  cap/None になり、確率本体は壊れない。
- **SC-004**: 推定単勝オッズ vs 実 odds の復元誤差(レース単位)と q の校正(NLL/Brier)が過去データで計測され、全出力が
  疑似評価として明示される。
- **SC-005**: 変換が p を一切使わず(市場オッズのみ)、p と q が別オブジェクト/列で区別される。
- **SC-006**: 決定論。複勝の頭数依存・小頭数縮退・該当なしが規則どおり。控除率が設定可能で logic_version に含まれる。

## Assumptions

- Feature 009(結合確率エンジン)が適用済み。`q` を入力として 009 を再利用する。
- 単勝オッズは `race_horses.odds`(確定=closing-oracle 疑似評価)または 008 の前売り単勝オッズ(より実運用寄り)。
  本フィーチャーは odds 源を問わず変換し、疑似性は odds 源に依存することを明示する。
- 控除率 JRA 既定(平成26年6月7日以降): 単勝/複勝 20%、馬連/ワイド 22.5%、馬単/三連複 25%、三連単 27.5%。時点依存で
  改定されうるため設定可能。
- `q` は市場の投票シェアであり、favorite-longshot bias を含む(真の勝率の校正済み値ではない)。
- 実 exotic オッズが無いため、評価は単勝復元・q 校正・結果校正に限る(実 exotic 価格の復元評価は将来、実 exotic オッズ
  取得後)。
- 複勝の `(1−takeout)/P(place)` は粗い近似(実払戻はプール分配・払戻対象頭数依存)。
- スキーマ変更なし。exotic EV/推奨・推定オッズの永続化・実 exotic オッズ取得は将来フィーチャー。日本語規約維持。
