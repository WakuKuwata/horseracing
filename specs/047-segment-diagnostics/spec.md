# Feature Specification: セグメント別モデル-市場診断 (Segment Diagnostics)

**Feature Branch**: `047-segment-diagnostics` / **Created**: 2026-07-02 / **Status**: Draft
**Input**: 「どの条件(距離/クラス/人気帯/新馬等)でモデルが市場に勝てる/負けるか」を可視化する診断。020 で全体 gap(p LogLoss 0.234 vs q 0.202)は判明済みだが**セグメント分解が無く**、運用判断(どの条件の買い目を信用するか)と次レバー発見(モデルが相対的に強い/弱い条件)の材料が無い。

## 背景と目的
製品目的は正直な意思決定支援(021)。市場 q が全体で p に勝つことは開示済みだが、gap は一様でないはず(仮説: 市場が情報を持ちにくい新馬・少頭数・穴人気帯では gap が縮む/逆転しうる — 026 の血統採用根拠と同型)。セグメント別の p vs q 品質比較を**事前登録した固定セグメント定義**で機械算出する診断 CLI を追加し、実データの結果を記録する。**診断は SECONDARY**(020 の規律継承): 採否ゲートや閾値調整に使わない・結果を見てセグメント定義を動かさない(憲法 III)。

## 事前登録セグメント定義(結果を見る前に固定・変更禁止)
- **surface**: 芝 / ダート / その他
- **dist_band**: 020 と同一(≤1400 sprint / ≤1800 mile / ≤2200 mid / >2200 long)
- **q_band(人気帯)**: q<0.05 / 0.05–0.15 / 0.15–0.30 / ≥0.30
- **race_class**: 新馬 / 未勝利 / OP系(オープン・G を含む) / 条件(その他)
- **field_size**: ≤8 / 9–13 / ≥14
- **debut**: 対象レースより厳密前に started 出走が無い馬(=新馬出走)か否か
- **指標**(セグメントごと): n / 実現勝率 / LogLoss(p) / LogLoss(q) / gap=LL(p)−LL(q)(正=市場優位) / mean p / mean q

## User Stories
### US1 - セグメント診断 CLI (P1)
運用者が walk-forward OOS の (p, q, win) をセグメント属性(DB の races/entries から race_id で結合、結果は勝敗ラベルのみ)で分解した表を得られる。
**AC**: (1) 事前登録定義どおりの決定論的な表(n/勝率/LLp/LLq/gap/平均) (2) 属性は races・厳密前 entries のみ(結果を診断入力にしない=win ラベルは既存 market_edge と同一) (3) 小 n セグメントは n を明示(隠さない) (4) market_edge と同じ PREDICTOR-AGNOSTIC(eval は training 非依存)

### US2 - 実データ所見の記録 (P1)
実 DB で診断を実行し、所見(どのセグメントで gap が大きい/小さい)を spec/research とメモリに記録する。
**AC**: 実行結果の表と解釈(SECONDARY・買い指示ではない注記付き)が記録される

## Requirements
- **FR-001**: セグメント定義・指標は本 spec の事前登録に完全一致し、実行後に変更しない(変更は別 feature で再登録)。
- **FR-002**: 診断は walk-forward OOS 予測(既存 market_edge と同一収集)に対して行い、in-sample を混ぜない。
- **FR-003**: セグメント属性の導出はレース静的属性(distance/track_type/race_class/started 頭数)と厳密前 entries(debut 判定)・市場 q のみ。**結果由来の属性を使わない**(win は評価ラベルとしてのみ)。
- **FR-004**: eval 層に置き training へ依存しない(020 の predictor-agnostic 規律)。診断出力は採否ゲートに接続しない(SECONDARY 明記)。
- **FR-005**: スキーマ・API・front 変更なし。CLI(training feature-diagnostic 系)からの実行。

## Success Criteria
- **SC-001**: 全サンプルが各軸で必ずいずれかのセグメントに入り Σn=全体 n(reconciliation)。
- **SC-002**: 同一入力で決定論(2 回実行で同一表)。
- **SC-003**: 属性リーク境界テスト(結果を変えても属性・セグメント割当が不変)。
- **SC-004**: 実 DB 実行の所見が記録される(SECONDARY 注記付き)。

## Assumptions
- 収集は market_edge の fold ループを流用(重い全期間再学習は許容、--from で範囲短縮可)。
- 表示(front)は deferred — 診断は運用者向け CLI+記録で足りる(製品表示に載せるなら別 feature で 021 規律から設計)。
## Deferred
front/API 表示・セグメント別校正(017 の条件別)・診断の自動定期実行

## 実データ所見(2026-07-02 実行、2021+ walk-forward OOS、n=181,341)
**SECONDARY 診断 — 買いシグナルではない。セグメント定義は事前登録どおり・実行後変更なし。**

| 所見 | 数値 |
|---|---|
| **全セグメントで市場 q が優位**(gap>0 が全行) | 全体傾向は 020 と一致。モデルが市場に勝つセグメントは存在しない |
| **gap 最小 = q_band 0.05–0.15(中穴)** | +0.0125 — モデルが市場に最も肉薄する帯 |
| **gap 最大 = 本命帯 q≥0.30** | **+0.2306**。市場 q(mean 0.405)は実現勝率 0.413 とほぼ完全一致、モデル p(mean 0.185)は**大幅過小**=モデル p は tail 圧縮 |
| 穴帯 q<0.05 | モデル p mean 0.055 vs 実現 0.0157 = **モデルは穴を約 3.5 倍過大評価**(FL バイアスの逆をモデルが持つ) |
| 仮説「新馬/少頭数で gap 縮小」は**棄却** | debut +0.0442 > non-debut +0.0359、small(≤8) +0.0597 > large(≥14) +0.0341。市場は新馬・少頭数でも(モデル比)むしろ強い |
| 芝ダ・距離帯・クラス | gap +0.034〜+0.045 で概ね一様(long でやや大 +0.0414、新馬クラス +0.0453) |

**解釈(運用判断向け)**:
1. モデル p の tail 圧縮(本命過小・穴過大)が gap の主因 → **046 の γ>1 シャープ化(実フィット γ=1.34)は方向として正しい**ことの独立傍証。一様 power では tail の非対称(本命側の乖離が穴側より大きい)を完全には吸収できない → 条件別/非対称校正は deferred 候補。
2. 意思決定支援としては「本命帯は市場 q を信用、モデルの独自見解が最も意味を持つのは中穴帯(0.05–0.15)」が正直な使い方。
3. 026 由来の仮説(市場はデビュー馬に弱い)は本指標では不支持 — 特徴として血統が効いたことと、セグメント優位は別物。
