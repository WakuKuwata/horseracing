# オッズ・疑似ROI仕様

## 基本方針

未来レースの購入判断では、予測確率から疑似オッズを算出し、市場オッズと比較して疑似ROIを算出する。

本プロジェクトでは、オッズはスナップショット保存しない。取得できた最新オッズを保存し、再取得時は最新値で上書きする。

## 用語

### 予測確率

モデルが出力する確率。

- `1着率`
- `2着以内率`
- `3着以内率`
- 券種別の組み合わせ確率

### 疑似オッズ

予測確率から算出する、公正オッズ相当の値。

初期案:

```text
疑似オッズ = 1 / 予測確率
```

実際には控除率、丸め、下限/上限、確率校正の扱いを後続設計で決める。

### 市場オッズ

netkeiba などから取得する現在の市場オッズ。

過去 JRA-VAN データでは、結果確定時の単勝オッズのみが存在する。

### 推定市場オッズ

実オッズが存在しない券種に対して、単勝オッズや予測確率から推定するオッズ。

2着以内、3着以内、連系券種の過去評価で必要になる。変換規則は後続設計で決める。

### 疑似ROI

ユーザーが未来レースの購入判断に使う指標。

初期案:

```text
疑似ROI倍率 = 予測確率 * 市場オッズ
疑似ROI率 = 疑似ROI倍率 - 1
```

市場オッズがない券種では、推定市場オッズを使って疑似ROIを算出する。

## 券種別の扱い

出走取消または競走除外が確定している馬は、すべての券種で計算対象から除外する。

除外後の出走馬だけを対象に、券種別確率、疑似オッズ、疑似ROI、おすすめ購入馬券を算出する。

### 単勝

- 確率: `1着率`
- 市場オッズ: 単勝オッズ
- 疑似ROI: `1着率 * 単勝オッズ`

### 複勝

- 確率: `3着以内率` を基本にする。
- 市場オッズ: 取得できる場合は複勝オッズを使う。
- 取得できない場合は、単勝オッズなどから推定市場オッズを作る。

### 馬連・ワイド・馬単・3連複・3連単

- 確率: `CombinationProbabilityEngine` が出力する券種別確率を使う。
- 市場オッズ: 取得できる場合は実オッズを使う。
- 取得できない場合は推定市場オッズを使う。
- 結合確率モデルと推定市場オッズの詳細は後続設計で決める。

## オッズ更新方針

- 最新オッズを取得したら既存値を上書きする。
- スナップショット履歴は保存しない。
- 最新オッズの `updated_at` は保持する。
- 予測や疑似ROIも、最新データに基づいて再計算し、必要に応じて最新結果で上書きする。

## 再現性の扱い

オッズスナップショット履歴は保存しないため、過去の任意時点の市場オッズを完全再現することは初期スコープ外とする。

ただし、予測・疑似ROI・おすすめ購入馬券を保存する場合は、その計算時に使用した市場オッズ、推定市場オッズ、疑似オッズ、疑似ROI、計算時刻、ロジックバージョンを結果レコードに保持する。これにより、最新オッズは上書きしつつ、提示済みの判断結果そのものは後から確認できるようにする。

## 注意事項

- 疑似ROIは購入判断用の指標であり、実払戻が存在しない券種では実現ROIそのものではない。
- 過去評価で実オッズが存在しない券種のROIは、変換規則に依存する疑似評価として扱う。
- 変換規則、閾値、買い目選定ロジックはバージョン管理する。
- オッズのスナップショット保存をしないため、過去の任意時点におけるオッズ差分監査は初期スコープ外とする。
