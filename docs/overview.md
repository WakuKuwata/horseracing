# プロジェクト概要

## 目的

過去レースデータから特徴量を作成し、予測モデルを学習し、未来に行われるレースの結果を予測する。

予測確率から疑似オッズと疑似ROIを算出し、市場オッズと比較できる形で提示することで、ユーザーが回収率の高い買い方を判断できる状態を目指す。

## MVP スコープ

- JRA-VAN の過去データと netkeiba の最新/未来レースデータを DB に取り込む。
- レース単位で出走馬の `1着率`、`2着以内率`、`3着以内率` を予測する。
- 予測確率から疑似オッズを算出する。
- 市場オッズまたは推定された券種別オッズと比較し、疑似ROIを算出する。
- 日付ごとのレース一覧とレース詳細で、予測・オッズ・疑似ROI・おすすめ購入馬券を確認できる。
- データ取り込み、予測、学習、モデル更新は初期は手動実行とする。

## 将来スコープ

- 枠順決定、馬体重確定、オッズ更新、レース直前などのタイミングで自動更新する。
- 単勝、複勝、馬単、馬連、ワイド、3連複、3連単の全券種に対応する。
- 血統データを embedding 化し、数値ベクトル特徴量として利用する。
- 予測根拠、危険な人気馬、妙味のある穴馬を表示する。
- RaceFront の native アプリ展開を見据える。

## 決定事項

- 仕様上の予測ラベル名は `1着率`、`2着以内率`、`3着以内率` を正とする。
- `win`、`place`、`show` などの英語名は、必要な場合だけ実装識別子として別途定義する。
- オッズはスナップショット保存しない。最新値を上書きし、画面・疑似ROI計算には最新値を使う。
- 疑似オッズと疑似ROIは、未来レースでユーザーが購入判断するための意思決定指標とする。
- 初期方針では、市場オッズを予測モデルの入力特徴量には使わない。予測確率を先に作り、オッズは疑似ROI算出で使う。市場オッズを特徴量に入れる実験を行う場合は、別設計でリーク防止と評価方法を定義する。
- JRA-VAN データは 2007 年以降を使用する。2006 年以前は ID 体系が異なるため使用しない。

## 成功指標

- `1着率`、`2着以内率`、`3着以内率` の予測品質が時系列評価で確認できる。
- 予測確率が実結果頻度に近いことを、校正指標で確認できる。
- 疑似ROIに基づく購入判断のバックテストで、ベースライン購入より高い回収率を示せる。
- 未来レースの詳細画面で、予測確率、疑似オッズ、市場オッズ、疑似ROI、おすすめ購入馬券を確認できる。
