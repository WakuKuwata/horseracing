# Market-aware betting policy 提案

## 目的

既存の `1着率` / `2着以内率` / `3着以内率` モデルはそのまま維持する。
その上で、回収率を上げることを目的に、オッズ・人気・市場 q を使って
「買う / 見送る / 賭け金を抑える」を判断する別レイヤーを追加検討する。

この提案は、オッズ入りの着率モデルを作るものではない。
既存モデルの予測確率 `p` を置き換えず、下流の買い目選定方策として扱う。

## 提案

既存モデルが出した `p` と、市場情報を組み合わせて、買い目候補ごとに次を出す。

- `bet / skip`
- `expected_return_score`
- `stake_multiplier`
- `edge_haircut`

入力候補は以下とする。

- 既存モデルの確率 `p`
- 市場 q
- 単勝オッズ
- 人気
- `p - q`
- `p / q`
- EV
- odds band
- 券種

## 券種ごとの扱い

過去データには基本的に単勝オッズしかないため、実回収として評価できるのはまず単勝に限定する。

| 券種 | 払戻の扱い | 評価上の扱い |
|---|---|---|
| 単勝 | 実単勝オッズを使う | real-return |
| 複勝 | 実払戻が無ければ推定オッズを使う | pseudo-return |
| 馬連・馬単・ワイド・三連複・三連単 | 実払戻が無ければ推定オッズを使う | pseudo-return |

2着・3着は単体の報酬として扱わない。
複勝、ワイド、馬連、三連系など、券種ごとの的中条件として扱う。

## シンプルな要件

1. 既存の着率モデル `p` を置き換えない。
2. オッズ・人気・市場 q は、買い目選定方策の入力としてのみ使う。
3. 単勝は実オッズで回収率を評価する。
4. 単勝以外は、実払戻データが無い限り疑似評価として扱う。
5. 学習・評価は時系列 walk-forward で行う。
6. `ROI > 1` だけで採用判断しない。
7. 採用判断では、現行の `p -> EV/Kelly` 方策と比較する。

## 避ける表現

- オッズ入りの着率モデル
- 市場情報込みの真の確率
- 疑似ROIで実回収率が上がった
- `p - q` がプラスだから買い
- 最終オッズで検証したので実運用でも再現できる

## 評価で見る指標

- 回収率
- 的中率
- bet 数
- 見送り率
- 最大ドローダウン
- 最大連敗
- log bankroll growth
- odds band 別の安定性
- fold 別の安定性

## 注意点

- 最終オッズを使った過去評価は、実運用より有利に見える可能性がある。
- 高オッズの偶然的中で回収率が大きく跳ねるため、単純な ROI だけでは判断しない。
- 推定オッズで評価した複勝・連系券種を、実回収率が出たとは扱わない。
- `p - q` がプラスだから買い、という単純ルールにしない。

## 次に決めること

- 単勝だけで先に検証するか。
- 複勝・連系券種の実払戻データを取得するか。
- 対象レースのオッズをどの時点で固定するか。
- 方策モデルの出力を将来 recommendations に保存するか。
