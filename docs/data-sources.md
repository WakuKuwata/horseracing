# データソース仕様

## JRA-VAN

過去レースデータの主データソースとして、ローカルファイルの JRA-VAN データを使用する。

このリポジトリでは `raw_data/jra-van/` 配下に年別データが存在するが、初期取込・学習・評価で使用するのは **2007 年以降** とする。

2006 年以前の JRA-VAN データは ID 体系が異なるため、初期スコープでは使用しない。過去分を使う場合は、別途 ID 正規化・マッピング仕様を作る。

取込コマンドやバッチ処理では、対象開始日を `2007-01-01` 以降に制限する。

### 利用目的

- 過去レース結果の取り込み。
- 学習用ラベルの作成。
- 馬、騎手、調教師、レース条件、出走情報、結果情報の蓄積。
- 結果確定時の単勝オッズを使った過去評価。

### 制約

- 過去レースデータには、結果確定時の単勝オッズのみ存在する。
- 2着以内、3着以内、連系券種の実オッズが存在しない場合は、単勝オッズなどから券種別オッズを推定する。変換規則は後続設計で決める。
- 2006 年以前は使用しない。

## netkeiba

最新のレース結果、未来レースの出馬表、最新オッズ、馬・騎手・調教師情報の補完に netkeiba を使用する。

netkeiba から取得したデータも最終的には DB に保存し、アプリケーションは DB を正として参照する。

### 利用目的

- 未来レースの一覧取得。
- 出馬表、枠番、馬番、騎手、馬体重、馬体重増減、人気、単勝オッズの取得。
- レース結果の更新。
- JRA-VAN に未登録の馬、騎手、調教師情報の補完。

### 更新方針

- 初期は任意タイミングの手動更新とする。
- 将来は枠順決定、馬体重確定、オッズ更新、レース直前、結果確定のタイミングで自動更新する。
- オッズはスナップショット保存せず、最新値で上書きする。

## ID 方針

- `raceId` は `YYYYVVKKDDRR` の 12 桁固定文字列とする。
- `raceId` は日付ではなく識別子として扱う。
- 馬、騎手、調教師の ID は JRA-VAN と netkeiba の対応関係を整理する必要がある。
- ID 対応表の有無、source-of-truth、衝突時の優先順位は未決事項とする。
