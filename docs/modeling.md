# モデル・特徴量仕様

## 予測対象

予測対象は以下の 3 ラベルを正とする。

- `1着率`
- `2着以内率`
- `3着以内率`

教師ラベルは `race_results.finish_order` から作成する。

各馬の出力確率は原則として以下を満たす。

- `0 <= 1着率 <= 2着以内率 <= 3着以内率 <= 1`
- レース内の `1着率` 合計は 1 に近い。
- レース内の `2着以内率` 合計は 2 に近い。
- レース内の `3着以内率` 合計は 3 に近い。

許容誤差、少頭数、取消、除外、同着、失格の扱いは後続設計で明文化する。

## 券種別確率

単勝、複勝、馬単、馬連、ワイド、3連複、3連単を計算対象とする。

`1着率`、`2着以内率`、`3着以内率` だけでは、順序付き・組み合わせ券種の確率は完全には決まらない。そのため、券種別確率を算出するための `CombinationProbabilityEngine` を抽象コンポーネントとして設ける。

具体的な方式は後続設計で決める。

候補:

- レース内順位スコアから着順分布を近似する。
- Plackett-Luce などの順位分布モデルを使う。
- Monte Carlo シミュレーションで着順分布を生成する。
- 券種ごとに個別の確率モデルを学習する。

## 特徴量方針

特徴量は過去レースデータ、出走馬情報、騎手、調教師、レース条件、血統情報から作成する。

初期モデルは LightGBM を使用する。

血統データは embedding を作成し、数値ベクトル特徴量として利用する。血統 embedding は単独で予測を担う主モデルではなく、補助特徴量として扱う。

## 過去レースがない出走馬の扱い

新馬など、過去にレースをしていない出走馬も予測対象に含める。

過去レースが存在しない馬については、過去成績由来の特徴量を推測で補完しない。該当特徴量は `Unknown` または `null` として扱い、過去出走数や特徴量利用可否フラグを明示する。

例:

- `career_starts = 0`
- `has_past_race = false`
- `past_performance_available = false`
- 過去3走平均着順、前走着順、前走上がり、前走距離、前走馬場などは `null`

`Unknown` と `0` は区別する。存在しない過去成績に `0` を入れると、実際には存在しない意味をモデルに与えるため禁止する。

予測時は、馬ごとに特徴量列を削るのではなく、固定特徴量スキーマを維持する。利用できない特徴量を欠損として渡し、利用可能な特徴量だけで予測できるようにする。

過去レースがない馬でも、以下のような発走前に利用可能な特徴量は使用する。

- レース条件。例: 開催場、距離、芝/ダート、馬場状態、クラス。
- 出走馬属性。例: 年齢、性別、斤量、枠番、馬番。
- 騎手、調教師。
- 血統情報と血統 embedding。
- 当日情報。例: 馬体重、馬体重増減。未発表の場合は欠損として扱う。

将来的には、履歴量に応じたセグメントを設ける。

- `debut`: 過去出走なし。
- `low_history`: 過去出走が少ない。例: 1 から 2 走。
- `standard`: 過去成績が十分にある。

初期は単一モデルで欠損・Unknown を扱い、必要に応じて `debut` / `low_history` 向けの補正、専用モデル、またはセグメント別校正を検討する。

## 出走取消・競走除外・競走中止の扱い

過去レースには、出走取消、競走除外、競走中止が存在する。これらは通常の完走結果とは区別し、特徴量として使用する。

### 過去レースでの扱い

出走取消・競走除外は実際にレースを走っていないため、実出走として扱わない。

- 過去出走数には含めない。
- 平均着順、平均着差、平均タイム、上がり、距離実績など、実走・完走を前提にする特徴量には含めない。
- `finish_order = 最下位` のような疑似着順に変換しない。
- 取消/除外の履歴特徴量としては使用する。

取消/除外の履歴特徴量例:

- 過去取消回数。
- 過去除外回数。
- 直近 N 走以内の取消/除外有無。
- 前走が取消/除外だったか。
- 取消/除外からの経過日数。
- 騎手・調教師単位の取消/除外率。

競走中止は実際に出走しているため、実出走として扱う。ただし完走していないため、完走を前提にする特徴量には混ぜない。

- 過去出走数には含める。
- 平均着順、平均着差、平均タイム、上がりなどの完走前提特徴量からは除外する。
- `finish_order = 最下位` のような疑似着順に変換しない。
- 中止履歴特徴量として使用する。

競走中止の履歴特徴量例:

- 過去中止回数。
- 中止率。
- 直近 N 走以内の中止有無。
- 前走中止フラグ。
- 中止からの経過日数。

### 未来レースでの扱い

未来レースの予測時点で、すでに出走取消または競走除外が確定している馬は予測対象に含めない。

除外対象:

- `1着率`、`2着以内率`、`3着以内率` の予測母集団。
- レース内確率の正規化対象。
- 券種別組み合わせ確率の計算対象。
- 疑似オッズ、疑似ROIの計算対象。
- おすすめ購入馬券の候補。

出走取消・競走除外の馬は画面上では状態を表示してよいが、予測確率や買い目候補は表示しない。

取消・除外を除いた残りの出走馬だけで、レース内確率を再正規化する。

競走中止は原則としてレース後に確定する結果であるため、未来レースの予測前に除外する状態としては扱わない。過去履歴特徴量として扱う。

## 利用可能タイミング

予測時点により利用できる情報が異なるため、特徴量には利用可能タイミングを定義する。

- 出馬表取得前。
- 枠順確定後。
- 馬体重確定後。
- オッズ取得後。
- レース直前。
- 結果確定後。

結果確定後の情報は学習ラベル・評価には使えるが、未来レース予測の特徴量として使ってはならない。

## リーク防止

- 学習・評価はレース開催順を保つ。
- walk-forward で分割する場合は日時で分割し、未来データが過去予測に混入しないようにする。
- JRA-VAN データは 2007 年以降のみを学習・評価対象とする。
- 特徴量の累積統計は、対象レースより前のデータだけで計算する。
- 血統 embedding やカテゴリ統計は train-only または out-of-fold で作成し、評価期間の結果を混入させない。
- 初期方針では、市場オッズを予測モデルの特徴量に含めない。オッズは予測後の疑似ROI算出で使う。市場オッズを特徴量に含める実験を行う場合は、別設計でリーク防止、利用可能タイミング、評価方法を定義する。

## 評価指標

予測品質:

- LogLoss
- Brier Score
- AUC
- NDCG
- 確率校正指標。例: ECE。

運用品質:

- 疑似ROI
- 回収率
- 的中率
- 最大ドローダウン
- 最大連敗数
- 見送り率

## モデル更新

初期は手動で学習・評価・モデル更新を実行する。

将来は定期的な再学習、精度劣化検知、再校正、モデル採用判定を自動化する。

モデル採用時は、既存モデルより予測品質または疑似ROIが改善していることを、同一評価条件で確認する。
