# 未決事項

このファイルは、後続で spec / plan 化するべき設計論点を管理する。

## P0: 券種別の結合確率モデル

`1着率`、`2着以内率`、`3着以内率` だけでは、馬単、馬連、ワイド、3連複、3連単の確率は一意に決まらない。

決めること:

- `CombinationProbabilityEngine` の方式。
- レース内順位分布の作り方。
- 券種別確率の正規化と整合性。
- 同一レース内での馬同士の相関の扱い。
- 計算量上限。

## P0: 推定市場オッズの変換規則

JRA-VAN の過去データには結果確定時の単勝オッズしかないため、2着以内、3着以内、連系券種の過去評価では推定市場オッズが必要になる。

決めること:

- 単勝オッズから複勝相当オッズを推定する規則。
- 単勝オッズから連系券種のオッズを推定する規則。
- 推定値を疑似ROIに使う場合の表示ラベル。
- 実オッズが存在する場合と推定オッズの場合の優先順位。

## P0: 予測・推奨テーブル

初期 DB は `aiuma` の core schema を参考にするが、予測結果と推奨結果の保存方式は未決。

決めること:

- `prediction_runs`
- `race_predictions`
- `recommendations`
- `model_versions`
- `feature_snapshots`
- `id_mappings`

オッズスナップショットは初期方針では作らない。

## P1: JRA-VAN と netkeiba の ID 対応

JRA-VAN と netkeiba の ID が常に一致するとは限らない。

決めること:

- 馬 ID の対応表。
- 騎手 ID の対応表。
- 調教師 ID の対応表。
- raceId の source-of-truth。
- ID 不一致時の手動修正フロー。

## P1: 特殊ケース

決めること:

- 過去出走なしの新馬。
- 過去出走が少ない `low_history` 馬。
- 取消。
- 競走除外。
- 失格。
- 同着。
- 競走中止。
- 少頭数レース。
- 海外馬、地方馬、未知血統。
- 馬体重未発表。
- オッズ未取得。

過去出走なし・低履歴馬については、単一モデルで扱うか、セグメント別モデル/補正/校正を行うかを後続設計で決める。

取消・除外・中止については、状態コード、DB 表現、特徴量名、結果取込時の正規化規則を後続設計で決める。

## P1: 買い目選定ロジック

- 購入単位。

- 的中率重視と回収率重視のバランス。
- 見送り条件。
- 買い目の重複やフォーメーション表現。
